# 杀戮尖塔

## 说明

使用GODOT引擎，制作一款卡牌肉鸽游戏。内容包括：

1. 一个开始菜单，选项包括继续游戏、新游戏、退出。

> 1.1. 如果没有存档，隐藏继续游戏按钮

> 1.2. 点击退出按钮后，弹出提示框：“确定要退出吗？”，选择是否。选择“是”后退出游戏。

> 1.3. 点击“新游戏”按钮时，如果有存档，弹出提示框：“确定要放弃吗？”，选择是否。选择“是”后清除存档。然后进入选人界面。

2. 游戏中，默认为房间，有查看牌组、查看地图、使用药水、菜单按钮。

> 2.1. 地图

>> 2.1.1. 地图分为可用和不可用两种状态。不可用状态下只可以查看地图，可用状态下可以点击一个房间来移动。

>> 2.1.2. 地图有一个生成函数。可以称为newMap。生成一连串房间，这将以按钮的形式显示在地图上。

>> 2.1.3. 地图有一个保存函数和读取函数，用来从存档中保存和读取当前的地图，而不丢失当前房间进度。

> 2.2. 牌组

>> 2.2.1. 游戏中，点击“查看牌组”按钮，可以查看角色的牌组。

>> 2.2.2. 牌组拥有增加和删除功能。

>> 2.2.3. 牌组在战斗初始化时，被战斗系统访问。

> 2.3. 战斗系统。作特别说明。

## 制作规划

建议你按照以下顺序逐步实现，每个阶段产出可运行的demo，避免一次性写太多导致混乱。

### 第一阶段：基础框架与存档菜单（1-2天）

搭建项目结构，创建 Autoload 脚本 SaveSystem, GameManager。

实现 GameData 资源类（包含牌组、地图、当前房间等字段，先留空）。

完成开始菜单场景，实现存档检测、新游戏/继续/退出逻辑。

完成选人界面（简易版，直接选择初始牌组）。

里程碑：可以新游戏并进入一个空房间场景。

### 第二阶段：房间与地图系统（2-3天）

创建房间场景，放置四个功能按钮。

实现地图生成器 MapGenerator：生成线性+分支的简易地图。

实现地图视图：用按钮网格显示所有房间，根据可用/不可用状态切换交互。

实现房间移动逻辑：点击可用房间，更新 GameData.current_room_id，保存游戏，并加载对应房间内容（先统一显示文字“进入战斗”）。

里程碑：可以在地图上移动，存档能保存当前房间位置。

### 第三阶段：牌组系统（1-2天）

定义 CardResource，创建3~5张基础卡牌（打击、防御等）。

实现牌组查看弹窗，显示当前牌组列表，支持删除卡牌（为后续功能预留）。

实现增加卡牌的方法（可以在房间添加一个测试按钮调用）。

里程碑：可以在游戏中查看和修改牌组，存档能保存牌组。

### 第四阶段：战斗系统核心（3-4天）

创建战斗场景，实现回合制框架（玩家回合/敌人回合切换）。

实现手牌区、抽牌堆、弃牌堆基础逻辑。

实现简单的卡牌效果（伤害、格挡）。

制作一个测试敌人，实现敌人意图和行动。

里程碑：可以从房间进入战斗，打完战斗胜利并返回房间，牌组变化能持久保存。

### 第五阶段：战斗系统完善与药水（2天）

增加能量系统，每回合回复能量。

实现药水系统（在房间界面使用药水恢复生命，战斗中也可用药水）。

增加战斗奖励（胜利后从几张卡牌中选择一张加入牌组）。

里程碑：完成一个可玩的战斗循环。

### 第六阶段：整合与打磨（2天）

将战斗胜利后的奖励、商店等事件与地图房间类型关联（例如战斗房间进入战斗，休息房间回复生命）。

增加音效、简单动画（卡牌移动、伤害数字）。

全面测试存档的完整性（战斗中断、房间切换等）。

发布一个可运行的游戏版本。
